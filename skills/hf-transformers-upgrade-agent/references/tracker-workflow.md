# Tracker Workflow

The tracker is the high-level state document for the upgrade.

It should record:

- source and target version
- stage overview
- current status of each stage
- links to runbooks, step notes, and validation artifacts

Use the tracker for:

- overall progress state
- stage status changes
- final “completed / deferred / ignored” conclusions

Do not overload the tracker with detailed patch narratives. Put step-level
detail into execution notes instead.
